  📊 TEST OBJECTIVE
═══════════════════════════════════════════════════════════════════════════════════

Goal: Determine the exact Byzantine fault tolerance limits of Sovereign Map
       by testing progressively higher percentages of malicious nodes across
       200+ federated learning rounds per configuration.

Success Criteria: Find the threshold where convergence FAILS


═══════════════════════════════════════════════════════════════════════════════════
  🎯 TEST SCENARIOS
═══════════════════════════════════════════════════════════════════════════════════

Total Configurations: 12 tests (6 Byzantine % × 2 aggregation methods)

┌─ BYZANTINE PERCENTAGES TESTED
│  ├─ 0% (Baseline, no malicious nodes)
│  ├─ 10% (7-8 Byzantine nodes out of 75)
│  ├─ 20% (15 Byzantine nodes)
│  ├─ 30% (22-23 Byzantine nodes) ← Theoretical BFT limit
│  ├─ 40% (30 Byzantine nodes) ← Beyond theoretical limit
│  └─ 50% (37-38 Byzantine nodes) ← Extreme adversarial case
│
├─ AGGREGATION METHODS TESTED
│  ├─ Median Aggregation (simple, robust)
│  └─ Multi-Krum Aggregation (advanced BFT algorithm)
│
└─ ROUNDS PER CONFIG: 200 rounds

═══════════════════════════════════════════════════════════════════════════════════
  📈 EXPECTED OUTCOMES
═══════════════════════════════════════════════════════════════════════════════════

BASELINE (0% Byzantine):
  ├─ Accuracy: 65% → 91%+ (smooth convergence)
  ├─ Loss: 3.5 → 0.1
  ├─ Convergence: ✅ EXCELLENT
  └─ Status: Reference point

0% Byzantine + Median:        ✅ Converges by round ~50
0% Byzantine + Multi-Krum:    ✅ Converges by round ~50

10% BYZANTINE:
  ├─ Median: Slight degradation, should still converge
  ├─ Multi-Krum: Detects outliers, minimizes impact
  └─ Expected: ✅ Both converge, Multi-Krum better

20% BYZANTINE:
  ├─ Median: Moderate degradation, slower convergence
  ├─ Multi-Krum: Handles well, maintains accuracy
  └─ Expected: ✅ Both converge (Multi-Krum faster)

30% BYZANTINE (Theoretical Max for BFT):
  ├─ Median: Significant degradation, ~85%+ accuracy
  ├─ Multi-Krum: Should handle, ~88%+ accuracy
  └─ Expected: ⚠️  Borderline, Multi-Krum more robust

40% BYZANTINE (Beyond Theoretical Limit):
  ├─ Median: ❌ Likely diverges or stalls at 75-80%
  ├─ Multi-Krum: ❌ Struggles, accuracy ceiling drops
  └─ Expected: FAILS CONVERGENCE

50% BYZANTINE (Extreme):
  ├─ Median: ❌ Complete failure, random walk
  ├─ Multi-Krum: ❌ Overwhelmed, no learning
  └─ Expected: ❌ BOTH FAIL

═══════════════════════════════════════════════════════════════════════════════════
  🔬 CONVERGENCE CRITERIA
═══════════════════════════════════════════════════════════════════════════════════

A configuration is considered CONVERGED if:

  ✅ Criterion 1: Final accuracy ≥ 80%
  ✅ Criterion 2: Last 10 rounds show <1% variance
  ✅ Criterion 3: Convergence rate stabilizes (delta < 0.5%)

A configuration FAILS if:

  ❌ Criterion 1: Final accuracy < 80% after 200 rounds
  ❌ Criterion 2: Accuracy oscillates or diverges in final 50 rounds
  ❌ Criterion 3: Loss increases in final 50 rounds (diverging)

For each of 200 rounds:
  • Model Accuracy (%) - Classification performance
  • Loss Value - Cross-entropy loss
  • Convergence Rate (delta %) - Accuracy change vs previous round
  • Byzantine Nodes Detected - Count of malicious nodes filtered
  • Aggregation Method Used - Median or Multi-Krum
  • Round Duration - Time to complete aggregation

═══════════════════════════════════════════════════════════════════════════════════
  🏗️  TEST INFRASTRUCTURE
═══════════════════════════════════════════════════════════════════════════════════

Node Configuration:
  • Total Nodes: 75 agents + 1 aggregator
  • Per-Node Memory: 31.12 MiB
  • Total Memory Used: ~2,400 MiB (29% of 8GB)
  • Headroom: ~5,700 MiB (70%) for testing

Aggregation Engine:
  • Median: Simple coordinate-wise median of all gradients
  • Multi-Krum: Selects k-closest gradients, Byzantine-resistant
  • Byzantine Detection: Outlier filtering based on distance
